# :rocket: Experiment Setup
Our experiment is structured into a source code, configutation and assets folder. The overall workflow can be controlled from within the `Makefile` at project root.<br>
In principle the workflow can be split into two parts: one for data generation and one for the actual simulation study

## Structure
### Source code (python)
We provide a `data` and a `model` module. In the first module we handle creation of datasets for the experimental setup, in the latter we organize the actual training ingredients. Due to different APIs of each ML algorithm we provide a `BaseParam` class within `params.py` that encapuslates different APIs and exposes a common framework for parameter handling. Similar for models/inducer algorithms we provide a `BaseInducer` class within `inducers.py`. The actual experiment is conducted in `main.py` (no seperate module here).
### Configuration
We have configurations for the synthetic data generation inside `conf/data`, configurations for the inducer algorithms inside `conf/inducer` and general configurations in `conf/main.yaml`.
#### inducer
The yaml file for inducers should match the name in the inducer registry of `inducers.py` since based on the yaml file name the actual inducer is instantiated (with parameters).<br>
Parameter values can be given as a list, e.g. "max_depth: [1,2,3]", as a single value ("max_depth": 1) or inside a "tune" header. In the latter case the parameter is optimized for each combination.
#### data
The simulation data is generated by registering a numpy resolver in order to write [numpy Generators](https://numpy.org/doc/stable/reference/random/index.html) directly as a configuration option.Additionaly we had to register a resolver for `equicov`. The parameters should match either a defined resolver of the argument as expected from numpy.
